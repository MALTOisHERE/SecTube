## 2025-03-06 - [CRITICAL] Mongoose Query Limit Unbounded & Negative Bypass (DoS)
**Vulnerability:** The API controllers accept an unbounded `limit` parameter directly from `req.query.limit` to use in database queries (e.g., `parseInt(req.query.limit) || 20`). A basic `Math.min(limit, 100)` allows an attacker to bypass the cap entirely by providing a negative integer (`?limit=-10000`), which Mongoose natively accepts (returning results in a single batch, causing DoS).
**Learning:** This missing and insufficient boundary check allows attackers to request massive datasets, forcing the backend and database to process and return enormous payloads, leading to resource exhaustion (CPU, memory, database load) and Denial of Service (DoS).
**Prevention:** Always enforce both a hard minimum and maximum boundary constraint for limit or pagination queries. Use a pattern like `Math.max(1, Math.min(parseInt(req.query.limit, 10) || DEFAULT, MAX_ALLOWED))` (e.g., capped at 100 with a floor of 1) before executing the query.
