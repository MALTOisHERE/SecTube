## 2025-05-24 - Rate Limiting on Authentication Endpoints
**Vulnerability:** Lack of rate limiting on sensitive authentication routes (`/api/auth/login` and `/api/auth/register`), making the application susceptible to brute-force attacks and account creation spam.
**Learning:** `express-rate-limit` is an effective middleware for mitigating these attacks at the application level. However, testing middleware that doesn't depend on the database can be tricky if the server crashes on DB connection failure. Isolating the server startup from the DB connection or mocking the DB connection is crucial for testing middleware in isolation.
**Prevention:** Always implement rate limiting on public-facing endpoints, especially those involving authentication or resource creation. Use distinct limiters for different actions based on their sensitivity and expected frequency. Ensure that the application can start or run in a limited mode even if external dependencies (like DB) are unavailable, to facilitate testing.
